# Глава 1: Введение в Go

## Что такое Go

Go (или Golang) — компилируемый статически типизированный язык программирования, разработанный Google в 2009 году.

**Ключевые особенности:**
- Компиляция в нативный машинный код (один исполняемый файл)
- Быстрая компиляция
- Встроенная поддержка конкурентности
- Сборка мусора (GC)
- Простой синтаксис

## Установка Go

1. Скачай установщик с официального сайта: https://golang.org/dl
2. Установи для своей ОС (Windows/Linux/macOS)
3. Проверь установку:
   ```bash
   go version
   ```
   Должно вывести что-то вроде: `go version go1.21.0 linux/amd64`

## Отличия от Java

| Аспект | Java | Go |
|--------|------|-----|
| **Компиляция** | В байт-код для JVM | В нативный машинный код |
| **Запуск** | Требует JVM | Самостоятельный исполняемый файл |
| **Скорость компиляции** | Медленная | Очень быстрая |
| **Размер результата** | Требует JRE | Один файл (включает runtime) |
| **Кросс-компиляция** | Через JVM автоматически | Через флаги GOOS/GOARCH |
| **Классы** | Есть | Нет (структуры + методы) |
| **Наследование** | Есть | Нет (композиция) |
| **Исключения** | Есть (try-catch) | Нет (возврат ошибок) |
| **Конкурентность** | Thread (тяжеловесные) | Goroutines (легковесные) |

## Структура Go программы

```go
package main

import "fmt"

func main() {
    fmt.Println("Hello, World!")
}
```

**Ключевые элементы:**

### package main
- Обязательное объявление для **исполняемых программ**
- Если пакет не `main`, то это библиотека (не запускается напрямую)
- Аналог `public class Main` с методом `main` в Java

### import "fmt"
- Импорт пакета из стандартной библиотеки
- `fmt` — пакет для форматированного ввода/вывода (format)
- В Go **неиспользуемые импорты** вызывают ошибку компиляции (в отличие от Java)

### func main()
- Точка входа в программу (аналог `public static void main(String[] args)`)
- В Go нет аргументов командной строки в сигнатуре (используй `os.Args`)

### fmt.Println()
- `Println` с **заглавной буквы** — экспортируемая функция (публичная)
- `println` с **маленькой буквы** — не экспортируемая (приватная)
- В Go видимость определяется регистром первой буквы, а не ключевыми словами `public/private`

## Важное отличие от Java: строгость компилятора

⚠️ **Go не скомпилируется, если:**
- Есть **неиспользуемые импорты**
- Есть **неиспользуемые переменные**
- Код не отформатирован по стандарту (можно автоматически исправить через `go fmt`)

В Java это просто warnings, в Go — ошибки компиляции. Это заставляет писать чистый код.

## Структура проекта

```
myproject/
├── go.mod          # Описание модуля (аналог pom.xml)
├── go.sum          # Хеши зависимостей
├── main.go         # Точка входа
└── utils/          # Другие пакеты
    └── helper.go
```

## Основные команды

- `go run main.go` — компиляция и запуск
- `go build` — создание исполняемого файла
- `go mod init module-name` — инициализация модуля
- `go mod tidy` — очистка зависимостей
- `go fmt` — автоформатирование кода

## Важно

✅ В Go есть **стандарт форматирования** — `go fmt` форматирует код единообразно
✅ Компиляция в **нативный код** = быстрый старт без JVM
✅ Один **исполняемый файл** без внешних зависимостей
